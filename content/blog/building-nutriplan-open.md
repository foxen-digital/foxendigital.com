# Building NutriPlan in the Open

Most meal planning apps have a fatal flaw: they assume one meal equals one day. NutriPlan doesn't. We built it to handle batch cooking, leftovers, and realistic meal planning—and we're building it entirely in the open.

## The Problem with Meal Planners

Existing meal planners force you to:

- Add the same recipe multiple times for batch cooking
- Manually track which meals are "to cook" vs "leftovers"
- Over-buy ingredients because shopping lists don't understand portions

NutriPlan solves this with intelligent serving tracking.

## Open Source Philosophy

We believe in:

1. **Transparency** — See how we build features
2. **Community input** — Shape the roadmap
3. **Quality demonstration** — Our code speaks for our skills

## What's Next

NutriPlan is our showcase product. Watch this space as we build out features and share what we learn along the way.
